## 1. 核心哲学

### 1.1 莱恩标注，系统学习
旧方法：系统硬编码参数猜测SC在哪 → 猜错→ 调参 → 还是猜错（盲人摸象）。
新方法：莱恩在图上标注正确答案 → 系统提取特征 → 积累后优化状态机参数（有靶子再射箭）。

**系统不猜、不自动生成。莱恩画什么，系统记什么。**

### 1.2 全息场域感知
K线图不是背景板，而是充满引力的场域。系统不仅提取标注点的数据，还要提取"空白处"的背景数据——未标注的震荡区域、宏观的真空区、趋势的健康度。

### 1.3 事件溯源：记录思维流
系统不保存静态的"最终画面"，而是保存一条时间线上的完整"动作流"：
```
[创建区间] → [标记SC] → [拖拽调整边界] → [删除SC] → [重新标记为再派发]
```
每一步操作都是学习素材。特别是**删除**——删除 = "假设被证伪" = 最宝贵的负样本。

### 1.4 完全复刻交易员手感
前端交互必须完全贴合莱恩在TradingView 中的原生习惯。不发明新工具，不引入技术术语。赋予基础画图工具（线段、通道、气泡）威科夫语义，系统在后台默默理解。

## 2. 与V3系统的关系

### 2.1 两份文档的定位

| 文档 | 定位 | 核心内容 |
|------|------|---------|
| SYSTEM_DESIGN_V3.md | 理论核心 | 三层架构、三大引擎、8种事件模板、55条设计决策 |
| WORKBENCH_DESIGN.md (本文档) | 数据采集入口 | 前端交互、特征提取、事件溯源、学习管道 |

### 2.2 数据流关系
```
莱恩在工作台标注↓
特征提取引擎采集数据
     ↓
转换为V3定义的 EventCase 结构
     ↓
写入V3记忆层（历史区间库 + 事件案例库）
     ↓
喂给V3进化引擎
     ↓
优化V3状态机的参数层和变体层
```

### 2.3 接口对齐
- **区间**：莱恩画的平行通道 →对齐V3的 `Range` 数据结构（channel_slope, channel_width, 三个锚点）
- **事件**：莱恩的气泡标注 → 对齐V3的 `Event` + `EventCase` 数据结构
- **阶段**：莱恩的垂直线划分 → 对齐V3的 `Phase` 枚举和 `PhaseTransition` 记录
- **结构线**：莱恩画的线段标注 "Creek"/"Ice" → 对齐V3的 `TrendLine` 数据结构
- **历史区间库**：莱恩画的每个区间自动注册为支撑/阻力 → 对齐V3的 `HistoricalRangeStore`

### 2.4 进化的对象
标注工作台不生成死规则（如"成交量>2x则SC"）。
进化的对象严格遵循V3的四层分离（RD-54）：

| 层级 | 是否进化 | 标注工作台的贡献 |
|------|---------|----------------|
| 不变层（区间逻辑、方向开关） | 否 | 无 |
| 参数层（阈值、权重） | 是 | 提供标注数据，用于优化参数 |
| 变体层（事件变体分类） | 是 | 提供多样化案例，用于聚类发现 |
| 策略层（交易决策） | 是 | 提供交易结果反馈 |

## 3. 前端交互规范

### 3.1 核心场域：平行通道 (Parallel Channel)

**交互方式**：选择"平行通道"工具 → 三次点击定义通道
- 第1点+ 第2点：定义主边界线（如SC到ST的连线）
- 第3点：定义对侧边界（如AR的位置），系统自动画出平行线

**视觉规范**：
- 通道内填充半透明背景色（约5%透明度）
- K线允许视觉刺穿通道边界（如Spring跌破下轨又拉回）
- 支持倾斜通道（上斜/下斜/水平）

**后台行为**：
- 系统自动计算 channel_slope（斜率）和 channel_width（宽度）
- 三个锚点坐标（时间戳+价格）写入数据库
- 通道自动注册到历史区间库（HistoricalRangeStore）

### 3.2 事件标记：气泡箭头 (Callout)

**交互方式**：选择"气泡箭头"工具 → 点击K线极值点（上方或下方）→ 输入文字（如"SC"、"Spring"、"这里扛住了"）→ 回车确认

**视觉规范**：
- 气泡框+箭头指向目标K线
- 事件标注在K线下方 = 买方事件（SC、Spring、SOS等）
- 事件标注在K线上方 = 卖方事件（BC、UTAD、SOW等）
- 莱恩自由决定标注位置，系统不强制

**后台行为**：
- 系统读取箭头指向的K线时间戳
- 触发语义理解层解析文字内容（见第6节）
- 触发特征提取引擎（见第4节）
- 系统基于极值点自动向左回溯到上一个事件，提取整段K线序列的量价特征

### 3.3 结构线：带内联文字的基础画线工具

**工具类型**：
- 水平线 (Horizontal Line)：固定价格水平
- 射线 (Ray)：从一点出发无限延伸
- 线段 (Line Segment)：两点之间
- 垂直线 (Vertical Line)：固定时间点

**内联文字功能**：
- 所有线段类工具的中间位置带有 "+" 号
- 点击 "+" 号可输入文字标注（如"Creek"、"Ice"、"Phase B"）
- 完全复刻 TradingView 的线段文字功能

**后台行为**：
- 系统自动提取线段的斜率（slope）和坐标
- 语义理解层解析文字，赋予威科夫物理意义：
  - "Creek" → 识别为吸筹区间的内部阻力线
  - "Ice" → 识别为派发区间的内部支撑线
  - "支撑"/"阻力" → 通用结构线
- 写入对应的 TrendLine 数据结构

### 3.4 阶段划分：垂直线 + 文本

**交互方式**：选择"垂直线"工具 → 点击阶段转折点 → 在"+" 号处输入阶段名称（如"Phase A"、"A"、"阶段B"）

**后台行为**：
- 系统以垂直线的时间戳作为阶段切割点
- 语义理解层将文字映射到 Phase 枚举（A/B/C/D/E）
- 创建 PhaseTransition 记录
- 阶段切割后，成交量基准自动重置（对齐V3 RD-22：成交量基准按阶段区分）

### 3.5 多周期同步

**核心原则**：大周期标注在小周期图表上必须保留可见。

**交互方式**：
- 顶部菜单提供时间级别切换（5m / 15m / 1H / 4H / 1D）
- 切换时间级别时，所有已有标注（区间、气泡、线段、垂直线）自动映射到新级别的坐标系
- 日线画的大区间在4H图上依然可见，支持"大周期定背景，小周期找入场"

**后台行为**：
- 每个标注记录其原始时间级别（timeframe字段）
- 渲染时根据当前查看级别动态转换坐标
- 不同级别的标注可以叠加显示

### 3.6 工具栏设计

极简工具栏，只保留莱恩需要的工具：

| 图标 | 工具 | 快捷键 | 用途 |
|------|------|--------|------|
| ▱ | 平行通道 | P | 画区间（核心） |
| 💬 | 气泡箭头 | T | 事件/文字标注 |
| ─ | 水平线 | H | 支撑/阻力水平 |
| ╱ | 线段 | L | Creek/Ice/趋势线 |
| → | 射线 | R | 延伸趋势线 |
| │ | 垂直线 | V | 阶段划分 |
| ✕ | 删除 | Del | 删除选中标注 |

## 4. 特征提取引擎

### 4.1 触发时机
当莱恩完成任何标注操作（创建气泡、画通道、划阶段）的瞬间，特征提取引擎自动启动。莱恩不需要任何额外操作。

### 4.2 回溯边界
特征提取的K线范围 = **从上一个已标注事件到当前事件之间的所有K线**。

示例：
```
... [SC标注] ←── 提取范围 ──→ [AR标注] ←── 提取范围 ──→ [ST标注] ...
```

如果当前事件是第一个标注（如SC），则回溯到图表可见范围的起点或趋势的起点。

### 4.3 基于三大定律的特征维度

#### 努力与结果定律 (Effort vs. Result)
寻找"吸收"与"背离"——大努力小结果意味着主力在吸收。

| 特征名 | 计算方式 | 含义 |
|--------|---------|------|
| effort_reward_ratio | K线实体长度 ÷ 成交量 | 量极大但实体小 =吸收 |
| rejection_ratio | 下影线长度 ÷ K线总振幅 | 多头把价格托起了多少 |
| effort_vs_result | V3公式: clamp(efficiency - 1.0, -1, +1) | -1=大努力小结果, +1=小努力大结果 |

#### 供求定律 (Supply and Demand)
寻找"动能衰竭"——供给还在涌出但需求已足以承接。

| 特征名 | 计算方式 | 含义 |
|--------|---------|------|
| price_deceleration | 当前K线跌幅 ÷ 前3根K线平均跌幅 | <1 = 减速（供求逆转） |
| body_overlap_pct | 连续K线实体重叠面积 ÷ 总实体面积 | 高重叠 = 单边供给减弱 |
| volume_trend | 回溯范围内成交量的线性回归斜率 | 正= 放量, 负 = 缩量 |

#### 因果定律 (Cause and Effect)
评估"势能释放"——跌得够不够深、时间够不够久。

| 特征名 | 计算方式 | 含义 |
|--------|---------|------|
| macro_drawdown | 从上一个显著高点到当前极值的总跌幅% | 势能释放程度 |
| time_dissipation | 这波趋势持续的K线数量 | 时间维度的因果积累 |
| support_proximity | 当前价格到最近历史支撑区间的距离% | 0= 精确触及, 远 = 悬空 |

### 4.4 多时间框架上下文 (MTF Context)

当莱恩在低级别（如4H）标注事件时，系统自动向上查询高级别的背景数据：

| 特征名 | 来源 | 含义 |
|--------|------|------|
| parent_trend_direction | 上一级别（如1D）的趋势方向 | 大背景是涨还是跌 |
| parent_phase | 上一级别当前所处阶段 | 大周期在Phase B还是D |
| distance_to_parent_support | 当前价格到上一级别最近支撑的距离 | 跨周期位置锚定 |
| mtf_resonance_score | 多级别方向一致性评分 | 共振程度 |

### 4.5 阶段转换速度 (Phase Velocity)

| 特征名 | 计算方式 | 含义 |
|--------|---------|------|
| phase_velocity | 阶段持续K线数 ÷ 历史平均阶段K线数 | <1 = 快速转换（如V型反转）|
| phases_skipped | 跳过的阶段列表 | 如[B, C] = 极端强势|

### 4.6 完整特征矩阵

每次标注产生一个特征向量，包含以上所有维度。特征矩阵随标注积累不断增长，成为进化引擎的训练数据。

## 5. 事件溯源架构

### 5.1 核心原则
系统记录莱恩在图表上的**每一个操作**，不仅是最终结果。操作流= 莱恩的思维流。

### 5.2 操作类型与学习价值

| 操作类型 | 学习价值 | 权重 |
|---------|---------|------|
| CREATE（创建） | 正向样本：莱恩认为这里是SC/AR/区间 | 高 |
| UPDATE（修改/拖拽） | 纠错边界：提取修正前后的特征差异 | 中 |
| DELETE（删除） | 负向样本：莱恩最初认为是SC但后来否定了 | 最高 |
| PERSIST（保留到最后） | 高置信度正确样本 | 最高 |

**删除的特殊价值**：
删除 = "假设被证伪"。系统记录删除前的完整特征快照，标记为负样本。
这些负样本是系统的"免疫抗体"——防止进化引擎在类似条件下犯同样的错。

### 5.3 操作记录数据格式

```python
@dataclass
class AnnotationAction:
    action_id: str                    # 唯一标识
    action_type: str                  # CREATE / UPDATE / DELETE
    timestamp: str                    # 操作时间（莱恩执行操作的时刻）
    
    # 操作对象
    target_drawing_id: str            # 被操作的Drawing ID
    target_type: str                  # "channel" / "callout" / "line" / "vertical"
    # 语义解析结果
    parsed_event_type: Optional[str]  # 语义理解层解析出的事件类型（如SC/AR）
    parsed_confidence: float# 莱恩表达的置信度（"确认SC"=1.0, "可能是SC"=0.5）
    raw_text: str                     # 莱恩输入的原始文字
    
    # 特征快照（操作发生时的市场状态）
    feature_snapshot: Dict            # 完整特征矩阵（见第4节）
    
    # 修改专用
    before_state: Optional[Dict]      # UPDATE时：修改前的Drawing状态
    after_state: Optional[Dict]       # UPDATE时：修改后的Drawing状态
    
    # 删除专用
    deleted_drawing_snapshot: Optional[Dict]  # DELETE时：被删除Drawing的完整快照
    deletion_context: Optional[Dict]  # DELETE时：删除时的市场状态（可能已经过了几根K线）
```

### 5.4 操作流时间线

所有操作按时间顺序存储为一条不可变的时间线（append-only log）：
```
[2026-04-0710:15] CREATE channel_001 — 平行通道 (ETH1D)
[2026-04-07 10:16] CREATE callout_001 — "SC" 指向 2024-03-15 K线
[2026-04-07 10:18] CREATE callout_002 — "AR" 指向 2024-03-22 K线
[2026-04-07 10:20] UPDATE channel_001 — 拖拽上轨调整宽度
[2026-04-07 10:25] DELETE callout_001 — 删除SC标注（莱恩改变判断）
[2026-04-07 10:26] CREATE callout_003 — "再派发" 指向 2024-03-15 K线
```

## 6. 语义理解层

### 6.1 设计目标
莱恩在气泡里写的是自然语言，系统需要理解他的意图。

### 6.2 三层理解机制

#### 第一层：关键词直接匹配
精确匹配标准威科夫术语，不区分大小写。

| 输入示例 | 匹配结果 |
|---------|---------|
| "SC" / "sc" | EventType.SC |
| "Spring" / "spring" | EventType.SPRING |
| "AR" | EventType.AR |
| "Phase B" / "阶段B" / "B" | Phase.B |
| "Creek" / "creek" | TrendLine (Creek) |
| "Ice" / "ice" | TrendLine (Ice) |

#### 第二层：语义推断
识别非标准但含义明确的表达。

| 输入示例 | 推断结果 | 推断逻辑 |
|---------|---------|---------|
| "恐慌抛售" | EventType.SC | 同义词映射 |
| "派发结束" | Phase.D完成 | 语义关联 |
| "这里扛住了" | 测试成功（ST/Spring SUCCESS） | 意图识别 |
| "假突破" | Spring或UTAD | 上下文判断 |
| "吸收" | 努力与结果背离 | 威科夫术语 |

#### 第三层：置信度感知
从莱恩的措辞中提取他的确定程度。

| 输入示例 | 置信度 | 学习权重 |
|---------|--------|---------|
| "确认SC" / "SC" | 1.0（高） | 正常权重 |
| "可能是SC" / "疑似SC" | 0.5（中） | 降低权重 |
| "观察中" / "待确认" | 0.2（低） | 仅记录不学习 |
| "注意这里的量" | N/A（纯笔记） | 不参与学习 |

####兜底规则
如果三层都无法解析，原样保存为纯文本笔记，不强行分类。

### 6.3 语义字典扩展
系统维护一个可扩展的语义字典（JSON格式），莱恩可以随时添加自己的习惯用语。

```json
{
  "同义词映射": {
    "恐慌抛售": "SC",
    "买入高潮": "BC",
    "假突破": ["SPRING", "UTAD"],
    "扛住了": "TEST_SUCCESS",
    "没扛住": "TEST_FAILED"
  },
  "置信度关键词": {
    "确认":1.0,
    "可能": 0.5,
    "疑似": 0.5,
    "观察": 0.2,
    "待定": 0.2
  }
}
```

## 7. 学习管道（标注→进化）

### 7.1 核心认知
学习的对象不是"SC长什么样"这种死规则。
学习的对象是**"在什么场域条件下，状态A会转换到状态B"**——即V3状态机的转换参数。

### 7.2 从标注到EventCase的转换

莱恩的一次标注操作 → 系统自动生成一个V3标准的EventCase：

```
莱恩标注 "SC" (气泡箭头指向某K线)↓
系统提取：- event: Event对象（type=SC,量价特征, 置信度）
  - range_snapshot: 当前区间状态快照
  - pre_sequence: 上一个事件到SC之间的K线序列
  - sequence: SC本身的K线（极值点）
  - post_sequence: SC之后的K线（随时间异步回填）
  - multi_tf_context: 高级别背景数据↓
写入V3记忆层的EventCaseStore
```

### 7.3 状态转换参数进化

莱恩标注的不是孤立事件，而是**事件序列**：
```
SC → AR → ST → Spring → SOS → JOC → MSOS → 趋势
```

每条完整序列教会系统：
1. **转换路径权重**：SC之后莱恩标了AR（而不是直接标Spring）→ 这条路径的概率权重+1
2. **跳过记录**：莱恩直接从SC跳到SOS，没有Phase B/C → 系统学会"V型反转"路径存在
3. **场域条件**：每次转换发生时的特征向量，作为"在什么条件下发生转换"的训练数据

### 7.4 变体发现

随着标注积累（如50个SC案例），系统可以进行聚类分析：
- 发现"莱恩标注的SC其实分两类：恐慌型（单根巨量长下影）和缓慢吸收型（多根渐进放量）"
- 这些变体自动注册到V3的变体层，为不同变体分配不同的检测参数

### 7.5 负样本的免疫学习

莱恩删除一个标注时，系统记录：
```
"在这个特定场域下，具备这些特征的K线，莱恩最初以为是SC，但后来证明不是。"
```
这些负样本形成"免疫抗体"——进化引擎在优化参数时，会同时最小化对负样本的误判率。

### 7.6 进化渐进路径（对齐V3 §9.1）

| 阶段 | 目标 | 标注数据需求 |
|------|------|------------|
| 阶段1：记录 | 完整记录所有标注和操作流，不做分析 | 开始积累 |
| 阶段2：统计聚合 | 计算每种事件的特征均值和分布 | 每种事件 ≥ 20个样本 |
| 阶段3：参数优化 | 用标注数据优化V3状态机的阈值参数 | 每种事件 ≥ 50个样本 |
| 阶段4：变体聚类 | 自动发现事件变体 | 每种事件 ≥ 100个样本 |

## 8. 数据结构定义

### 8.1 Drawing（画图对象）

```python
@dataclass
class Drawing:
    drawing_id: str                   # 唯一标识
    drawing_type: str                 # "channel" / "callout" / "horizontal_line" / "line_segment" / "ray" / "vertical_line"
    points: List[AnchorPoint]         # 锚点列表（1-3个点）
    symbol: str                       # 如 "ETHUSDT"
    timeframe: str                    # 如 "1d"
    
    # 时间戳
    created_at: str                   # 创建时间
    modified_at: str                  # 最后修改时间
    
    # 莱恩输入的原始信息
    raw_text: Optional[str]           # 气泡/内联文字的原始内容
    
    # 语义理解层解析结果
    parsed_type: Optional[str]        # 解析出的威科夫类型（EventType/Phase/TrendLine类型）
    parsed_confidence: float          # 莱恩表达的置信度
    
    # 视觉属性
    color: Optional[str]
    line_style: Optional[str]         # solid / dashed / dotted
    line_width: Optional[int]
    fill_opacity: Optional[float]     # 通道填充透明度
    
    # 自动提取的特征
    auto_features: Optional[Dict]     # 特征提取引擎的输出
```

### 8.2 与V3数据结构的映射关系

|工作台Drawing | V3数据结构 | 映射逻辑 |
|--------------|-----------|---------|
| channel (平行通道) | Range | 三个锚点→primary_anchor_1/2 + opposite_anchor,斜率→channel_slope |
| callout (气泡箭头) | Event + EventCase | parsed_type→event_type, 箭头指向→sequence K线, 特征→EventCase字段 |
| line_segment/ray (线段/射线) + "Creek"/"Ice" | TrendLine |两点→slope+intercept, 文字→Creek或Ice |
| vertical_line (垂直线) + "Phase X" | PhaseTransition | 时间戳→切割点, 文字→Phase枚举 |
| horizontal_line (水平线) |通用支撑/阻力 | 价格→support/resistance level |

### 8.3 存储格式

与V3记忆层格式保持一致，使用JSON文件：
```
wyckoff/data/annotations/
├── ETHUSDT/
│   ├── drawings.json          # 所有Drawing对象
│   ├── action_log.json        # 操作流时间线（append-only）
│   └── features/
│       ├── SC_features.json   # SC类标注的特征矩阵
│       ├── AR_features.json   # AR类标注的特征矩阵
│       └── ...
```
