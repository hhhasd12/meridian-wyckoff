### 2. 功能职责划分

#### 2.1 进化系统 (Evolution System)
**职责**：策略参数优化、变异生成、WFA验证
**组件**：
- `StrategyOptimizerAgent` - 策略优化Agent
- `SelfCorrectionWorkflow` - 自修正工作流
- `WFABacktester` - WFA验证器
- `WeightVariator` - 权重变异器

**输入**：当前配置、性能评估结果
**输出**：优化后的配置、验证结果

#### 2.2 回测系统 (Backtest System)
**职责**：策略回测、性能评估、交易模拟
**组件**：
- `BacktestValidatorAgent` - 回测验证Agent
- `BacktestEngine` - 回测引擎
- `PerformanceMonitor` - 性能监控

**输入**：策略配置、历史数据
**输出**：回测结果、性能指标

#### 2.3 监控系统 (Monitor System)
**职责**：实时状态监控、进度展示、日志记录
**组件**：
- `ReportAgent` - 报告Agent
- `WebDashboard` - Web界面
- `PerformanceMonitor` - 性能监控

**输入**：系统事件、任务状态
**输出**：可视化界面、报告文档

## 二、系统核心功能分析

### 1. 用户需要看到的内容

#### 1.1 多周期状态监控
- **6个时间周期的实时状态**
  - 1d, 8h, 4h, 1h, 15m, 5m
  - 每个周期的K线数量
  - 时间范围
  - 当前威科夫阶段
  - TR边界（高/低）
  - 趋势方向

#### 1.2 进化进度监控
- **当前任务状态**
  - 任务类型（进化/回测/WFA验证）
  - 进度百分比
  - 预计剩余时间
  - 当前处理的时间周期

- **进化历史记录**
  - 优化ID
  - 改进百分比
  - WFA验证结果
  - 时间戳

#### 1.3 性能指标
- **核心指标**
  - 夏普比率
  - 最大回撤
  - 胜率
  - 盈亏比
  - 总收益

- **多周期加权分数**
  - 每个周期的单独分数
  - 冲突解决结果
  - 主要偏向

#### 1.4 交易信号
- **买入信号**
  - 时间戳
  - 价格
  - 置信度
  - 来源周期

- **卖出信号**
  - 时间戳
  - 价格
  - 置信度
  - 来源周期

### 2. 用户需要控制的内容

#### 2.1 进化参数
- **周期数**：运行多少个进化周期
- **评估模式**：完整/快速/自定义
- **评估K线数**：使用多少数据进行评估

#### 2.2 系统控制
- **暂停/继续**：暂停当前任务
- **重置系统**：清除所有状态
- **导出结果**：保存优化结果

## 二、性能优化方案

### 1. 数据采样优化

#### 1.1 智能采样策略
```
完整模式：使用全部数据（适合最终验证）
快速模式：使用最近1000条K线（适合快速迭代）
自定义模式：用户指定K线数量
```

#### 1.2 周期权重采样
```
高权重周期（1d, 4h）：使用全部数据
低权重周期（15m, 5m）：使用采样数据
```

### 2. 计算优化

#### 2.1 并行计算
- 多周期并行评估
- 使用多进程/多线程

#### 2.2 缓存机制
- 缓存Context构建结果
- 缓存状态机中间状态

#### 2.3 增量计算
- 只计算新增K线
- 复用之前的计算结果

### 3. WFA优化

#### 3.1 减少WFA窗口数
- 默认10个窗口 → 可调整为3-5个

#### 3.2 快速WFA模式
- 只验证关键指标（夏普比率）
- 跳过次要指标验证
