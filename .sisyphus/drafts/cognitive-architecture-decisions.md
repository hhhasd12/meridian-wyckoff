## 一、问题诊断（已确认）

### 核心问题：认知顺序反了

当前V4状态机从第一根K线就开始找威科夫事件（PS/SC/PSY/BC），完全不管"当前是趋势还是区间"。

正确的认知顺序：**先判断市场状态 → 再假设结构类型 → 再在结构内找事件**

### 代码层面的具体问题

1. **无区间概念**：TRDetector算的是统计包络线（CurveBoundaryFitter枢轴点拟合），不是威科夫意义上的SC_LOW~AR_HIGH区间。状态机的critical_levels记录了SC_LOW/AR_HIGH，但从未用于计算position_in_tr
2. **冷启动失败**：第一根K线就进入检测模式，PSYDetector门槛极低（0.2），任何阴线+放量就触发派发结构
3. **B阶段太短**：没有最小停留时间，B阶段6-13根K线就被跳过（应占结构40-60%）
4. **BoundaryManager是空壳**：已实现完整的propose→lock→invalidate生命周期，但state_machine_v4.py从未使用它
5. **plugin.py是死壳**：engine.py直接import并使用WyckoffStateMachineV4，plugin.py的接口适配问题不影响生产运行（但需要清理）
6. **transition_history bug**：`_confirm_and_advance()`中先更新last_confirmed_event再记录历史，导致from==to

## 二、核心设计共识

### 共识1：区间是假设性的，不是确认性的

- 不存在"100%确认这是区间"的时刻
- 只有"概率够高了可以开始行动"的时刻
- 停止行为 → AR → ST不破 = 置信度够高，可以在假设区间内操作
- 后续事件持续验证 → 置信度上升
- 结构破坏（如跌破SC_LOW）→ 假设失败

### 共识2：区间由事件定义，不由独立检测器定义

- SC定义初始下沿，AR定义初始上沿
- ST精化下沿（SC→ST连线可能有斜率）
- 区间 = SC→ST连线 + 平行线经过AR附近
- 区间可以上斜（底部抬升=吸筹信号更强）、水平、下斜

### 共识3：需要结构级假设，不只是事件级假设

当前V4只有事件级假设（"这根K线是不是ST"）。
缺少结构级假设（"这里正在形成一个吸筹结构，当前置信度0.6"）。

结构假设应该：
- 在第一个停止行为出现时创建（低置信度）
- 随事件序列推进，置信度逐步上升
- 置信度影响交易信号强度
- 结构破坏 → 假设否定

### 共识4：趋势是独立运行模式，不是和SC/AR平级的状态

三个运行模式：
- **TRENDING**：只监听停止行为，不跑区间内检测器
- **TRANSITIONING**：停止行为出现后，等待AR/ST来形成区间假设
- **RANGING**：在假设区间内跑事件序列（B→C→D→E）

### 共识5：冷启动 = 无状态观察期

- 不是固定N根K线预热
- 而是等第一个停止行为出现才开始
- 统计数据（均量、价格历史等）始终在积累
- 冷启动主要影响进化/回测场景，实盘有完整历史

### 共识6：打分系统服务于概率判断

- 三大原则（供需/因果/努力结果）已实现
- 问题不是"没有"，而是打分结果没用在结构级别判断上
- 需要把单事件打分汇聚到结构置信度

## 三、待定决策（需要威科夫理论约束）

### 待定1：每个阶段的必要/充分条件

- A阶段（PS/SC/AR/ST）：哪些事件是必须出现的？
- B阶段：持续时间的理论下限？
- C阶段（SPRING/UTAD）：必须出现还是可选？
- D阶段（SOS/SOW）：和B阶段的区分标准？

### 待定2：结构失败的完整判定标准

- 跌破SC_LOW多少算失败？（当前用3%）
- 还有哪些情况算结构失败？
- 失败后应该怎么办？（回TRENDING？开始新结构？）

### 待定3：TRANSITIONING的完整路径

- 停止行为 → AR → ST不破 → 区间假设（已定）
- 停止行为 → AR → ST破了 → 然后呢？
- 停止行为 → 没有明显AR → 然后呢？
- 这些路径需要威科夫理论指导

### 待定4：区间斜率的处理

- SC→ST连线斜率怎么算？需要几个点？
- 平行线怎么定位到AR附近？
- 斜率信息怎么影响后续判断？
