# 威科夫理论约束规则 — 状态机设计权威参考

> **用途**：本文档从三本威科夫原著精炼版（83个文件）中提取所有对状态机有约束力的规则
> **来源**：孟洪涛《威科夫操盘法》+ 安娜·库林《量价分析》+ 威科夫《擒莊秘笈》
> **日期**：2026-03-23
> **状态**：完成

---

## 一、公理层 — 三大定律（不可违反）

### L-1 供求定律
- 需求 > 供给 → 价格上升；供给 > 需求 → 价格下跌
- **这是价格运动的唯一根本原因**，所有事件识别都必须回到供求逻辑

### L-2 因果定律
- 因果呈正比例关系：横向整理时间越长（因），后续趋势越强（果）
- 吸筹/派发区间不够长 → 后续SOS/SOW可信度应降低
- **状态机约束**：区间持续时间/宽度是结构置信度的关键输入

### L-3 努力与结果定律
- 成交量=努力，价格变动=结果，两者必须匹配
- 放量不涨/不跌 = 异常 = 存在反向力量 = 状态转换信号
- **状态机约束**：每个事件判定必须同时检查量价一致性

### 量价不可分离公理
- 离开价格谈成交量毫无意义（V-2）
- 没有成交量验证，价格行为无效（V-5）
- 成交量是相对的，必须与历史正常水平对比（V-1）
- 成交量是唯一不可隐藏的主力行为痕迹（V-4）

## 二、事件定义 — 吸筹阶段（精确特征+量价约束）

### PS（Preliminary Support，初次支撑）
- **定义**：熊市末期第一次明显买入行为，阻止下跌
- **价量约束**：成交量放大但价格不再创新低
- **作用**：预示长期下跌趋势可能结束。PS只是第一个警示信号，**不是买入信号**
- **状态机约束**：PS是吸筹序列的第一个事件，必须在下跌趋势之后出现

### SC（Selling Climax，恐慌抛售/卖出高潮）
- **定义**：恐慌性抛售，超大成交量+大幅下跌，供应集中释放
- **价量约束**：成交量必须是"异常放大"级别（相对历史），价格急剧下跌形成尖底
- **结构约束**：**确立吸筹区间的下边界**（SC_LOW）
- **核心价值**：快速消耗供应，创造供应短缺的底部环境
- **时序约束**：必须在PS之后出现
- **不能在SC后立刻抄底**：必须等待吸筹过程完成的信号

### AR（Automatic Rally，自动反弹）
- **定义**：SC后的强劲反弹，抛压枯竭后的自然回升
- **结构约束**：**确立吸筹区间的上边界**（AR_HIGH）
- **时序约束**：必须紧跟SC之后
- **特性**：无需特殊量能要求，是自然回升
- **区间形成**：AR_HIGH和SC_LOW构成后续震荡区间的上下边界

### ST（Secondary Test，二次测试）
- **定义**：价格回落测试SC低点区域
- **价量约束**：**成交量必须明显低于SC**（核心验证条件）
- **成功条件**：价格回落到SC区域附近，成交量明显低于SC → 供应枯竭确认
- **失败条件**：如果ST成交量不小于SC → 底部不可靠
- **数量**：可以出现**多个ST**（吸筹中期反复震荡）
- **连续ST约束**：每次ST的成交量应呈递减趋势

### Creek（小溪/阻力线）
- **定义**：吸筹区间内的阻力线，连接多个反弹高点
- **画线方法**：连接ST之后多次反弹的高点
- **结构约束**：是SOS最终突破的参考线

### Spring（弹簧/假突破下方）
- **定义**：价格短暂跌破SC低点/震荡区间下边界，随后快速收回区间内
- **四个必要条件（全部必须满足）**：
  1. 已经看到SC、AR、ST，横盘一段时间
  2. 价格跌破震荡区间下边界
  3. **跌破时成交量小于SC**（确认供应枯竭）
  4. 迅速反弹回到区间内（需求进场）
- **有效确认三要素**：低量跌破 + 迅速反弹 + 反弹放量
- **Spring三分类**：
  - **1号Spring**：跌破支撑但成交量大 → **风险高**（不理想）
  - **2号Spring**：跌破支撑成交量小 → **理想的买入信号**
  - **3号Spring**：仅测试支撑但未跌破 → 较弱的买入信号
- **幅度约束**：跌破幅度通常**不超过区间高度的10%**
- **速度约束**：收回速度快，**当日或次日即收回**
- **时序约束**：必须在多个ST之后、吸筹末期出现
- **目的**：洗出最后的弱手，吸收廉价筹码

### LPS（Last Point of Support，最后支撑点）
- **定义**：Spring后的回调低点
- **价量约束**：**成交量比Spring时更低**
- **价格约束**：价格不再创新低，形成**更高的低点**（higher low）
- **确认意义**：确认吸筹完成
- **时序约束**：必须在Spring之后出现

### SOS（Sign of Strength，强势信号）
- **定义**：长阳线+大成交量，接近或突破Creek/震荡区间上边界
- **价量约束**：**必须放量**
- **确认条件**：SOS之后的回调必须是短K线+低成交量（供应没有跟随）
- **SOS回调放量 → SOS可能是假信号**
- **时序约束**：必须在LPS之后出现

### JOC（Jump Over the Creek，跳离震荡区）
- **定义**：放量突破震荡区间上边界/Creek
- **价量约束**：**突破时成交量必须放大**
- **K线特征**：跳空或长阳
- **确认意义**：牛市正式开始的标志
- **前提**：SOS和LPS已经出现

### BUEC（Back Up to the Edge of the Creek，回测）
- **定义**：JOC后回调到原阻力位（Creek）
- **逻辑**：阻力变支撑
- **理想回测**：回调时成交量萎缩，说明供应稀少，是加仓好时机

## 三、事件定义 — 派发阶段（精确特征+量价约束）

### PSY（Preliminary Supply，预备供应）
- **定义**：牛市末期第一次明显卖出行为，阻止上涨
- **价量约束**：成交量放大但价格不再创新高
- **作用**：预示长期上涨趋势可能结束

### BC（Buying Climax，抢购高潮/买入高潮）
- **定义**：狂热买入，超大成交量+大幅上涨，需求集中释放
- **价量约束**：成交量爆发（创新高），价格急剧上涨，但涨幅收窄或收长上影线
- **结构约束**：**确立派发区间的上边界**（BC_HIGH）
- **可选性**：不是所有派发都有BC，有些是渐进式派发（边拉边出）

### AR（Automatic Reaction，自动回落）
- **定义**：BC后的第一次回调，买盘枯竭后的自然回落
- **结构约束**：**确立派发区间的下边界**（AR_LOW）
- **时序约束**：必须紧跟BC之后

### ST（Secondary Test，二次测试-顶部版）
- **定义**：反弹测试BC高点区域
- **价量约束**：**成交量应明显低于BC**
- **确认意义**：确认买盘已枯竭
- **数量**：可出现多个ST

### Ice（冰线/支撑线）
- **定义**：派发区间内的支撑线，连接多个回调低点
- **结构约束**：是SOW最终跌破的参考线
- **跌破后**：变为阻力（支撑变阻力原则）

### UTAD（Upthrust After Distribution，上冲回落）
- **定义**：价格突破BC高点/区间上边界后迅速回落，假突破
- **三个特征（同时出现确认有效）**：
  1. 突破时成交量大（吸引追高资金）
  2. 突破后迅速回落到区间内（主力无继续拉升意愿）
  3. 回落时成交量也大（供应涌出）
- **幅度约束**：突破幅度通常**不超过区间高度的10%**
- **速度约束**：**当日或次日即回落**
- **镜像关系**：UTAD是派发版的Spring
- **目的**：诱多，吸引最后的买盘接盘
- **可选性**：有些派发没有UTAD，直接SOW

### SOW（Sign of Weakness，弱势信号）
- **定义**：长阴线+大成交量，接近或跌破Ice/震荡区间下边界
- **价量约束**：**必须放量**
- **确认条件**：SOW之后的反弹必须是无力的（短K线+低成交量）
- **SOW反弹强劲 → SOW可能是假信号**

### LPSY（Last Point of Supply，最后供给点）
- **定义**：SOW后的反弹高点
- **价量约束**：**成交量比SOW时更低**，反弹幅度小于SOW下跌幅度
- **价格约束**：价格不再创新高，形成**更低的高点**（lower high）
- **确认意义**：确认派发完成，是做空的最佳时机
- **镜像关系**：LPSY和LPS是镜像（LPS后做多，LPSY后做空）

### 派发第二阶段特征（高位震荡出货期）
- 价格在BC和AR形成的区间内高位震荡
- **放量滞涨**：每次上涨伴随放量但涨幅有限或无法创新高
- 每次回调成交量也不小，供应持续释放
- 时间特征：**可能持续数周到数月**
- 核心信号：放量不涨=供应在持续释放=主力在出货

### 没有BC的渐进式派发
- 上涨趋势逐渐减速，没有放量冲顶
- 高点越来越低，价格无法创新高
- 成交量在横盘区间放大但价格滞涨
- **更难识别**，散户容易误认为"正常整理"

## 四、阶段序列约束（时序规则）

### 吸筹完整序列（严格时序）
```
下跌趋势 → PS → SC → AR → ST(多个) → [横向震荡/Creek形成] → Spring → LPS → SOS → JOC → BUEC → 上涨趋势
```
- PS→SC→AR→ST 是A阶段（停止下跌）
- 横向震荡 是B阶段（吸筹主体，占结构40-60%时间）
- Spring/LPS 是C阶段（最终测试）
- SOS/JOC/BUEC 是D阶段（开始上涨）
- **核心事件不可或缺**：SC、Spring、SOS、JOC（序列可能不完整，但这四个是核心）

### 派发完整序列（严格时序）
```
上涨趋势 → PSY → BC → AR → ST(多个) → [横向震荡/Ice形成] → UTAD → LPSY → SOW → 下跌趋势
```
- PSY→BC→AR→ST 是A阶段（停止上涨）
- 横向震荡 是B阶段（派发主体）
- UTAD/SOW 是C阶段（确认派发结束）
- 跌破支撑 是D阶段（开始下跌）
- **核心事件不可或缺**：BC（或渐进式派发）、UTAD或SOW、LPSY

### 完整市场周期八阶段（严格顺序不可跳跃）
```
吸筹 → 买入高峰 → 供给测试 → 上涨趋势 → 派筹 → 抛售高峰 → 需求测试 → 下跌趋势 → 循环
```
- 阶段顺序严格不可跳跃
- 测试阶段（供给测试/需求测试）是**门控点**——只有测试成功（低成交量）才能进入下一阶段
- **俄罗斯套娃**：此周期存在于任意时间跨度，小周期嵌套在大周期内

### 序列约束规则汇总
1. PS/PSY必须在趋势末期出现（分别在下跌末期/上涨末期）
2. SC/BC确立区间边界后，才能出现ST
3. ST必须在AR之后出现
4. Spring/UTAD必须在多个ST之后的末期出现
5. LPS/LPSY必须在Spring/UTAD之后出现
6. SOS/SOW是最终确认事件，必须在LPS/LPSY之后
7. **惯性约束**：市场如油轮，不会嘎然停止又迅速启动。状态转换需要"扫尾工作"
8. **不对称性**：上涨缓慢渐进，下跌快速剧烈（"走楼梯上楼，坐电梯下楼"）
9. 派发阶段时间通常比吸筹阶段短（主力在顶部更急于出货）

### 吸筹与派发的镜像关系
| 吸筹 | 派发 |
|------|------|
| PS | PSY |
| SC（卖出高潮） | BC（买入高潮） |
| AR（自动反弹） | AR（自动回落） |
| ST（二次测试底部） | ST（二次测试顶部） |
| Spring（向下假突破） | UTAD（向上假突破） |
| LPS（最后支撑点） | LPSY（最后供给点） |
| SOS（强势信号） | SOW（弱势信号） |
| JOC（跳离震荡区） | 跌破Ice |
| BUEC（回测支撑） | 回测阻力 |

## 五、量价验证规则（事件有效性判定）

### 成交量递减规则（验证供应/需求枯竭）
| 规则ID | 约束 | 判定标准 |
|--------|------|----------|
| VOL-01 | ST量 < SC量 | 底部ST验证 |
| VOL-02 | ST量 < BC量 | 顶部ST验证 |
| VOL-03 | LPS量 < Spring量 | 最后支撑点验证 |
| VOL-04 | LPSY量 < SOW量 | 最后供给点验证 |
| VOL-05 | Spring跌破时量必须小 | 供应枯竭确认 |
| VOL-06 | 连续ST量逐次递减 | 供应/需求逐步枯竭 |

### 放量规则（验证需求/供应存在）
| 规则ID | 约束 | 判定标准 |
|--------|------|----------|
| VOL-07 | SOS必须放量 | 长阳线+大成交量 |
| VOL-08 | JOC必须放量突破 | 突破Creek |
| VOL-09 | SOW必须放量 | 长阴线+大成交量 |
| VOL-10 | 死角突破必须放量 | 确认突破有效 |

### 努力与结果匹配规则
| 规则ID | 模式 | 含义 |
|--------|------|------|
| ER-01 | 大成交量+窄振幅 | 停止行为（努力无结果） |
| ER-02 | 放量滞涨 | 派发信号（供应吸收需求） |
| ER-03 | 放量不跌 | 底部信号（需求吸收供应） |
| ER-04 | 高成交量+低实体 | 力量被吸收（供需转换中） |
| ER-05 | 低成交量+高实体 | 陷阱（价格无真实交易支撑） |

### 趋势秩序量价规则
- **健康上涨**：上涨放量+回调缩量 → 秩序完整 → 继续持有
- **健康下跌**：下跌放量+反弹缩量 → 秩序完整
- **秩序被打破**：回调放量 或 上涨缩量 → 趋势终止警示
- **量价确认/异常二元判定**：每根K线只有两种结果——被成交量确认（继续），或存在异常（变化信号）

### 趋势不同阶段的成交量特征
- 趋势初期：成交量温和放大（资金开始进入）
- 趋势中期：成交量稳定或略有递减（浮筹已洗净）
- 趋势末期：成交量急剧放大（高潮）= 最后一批资金涌入 = 趋势即将结束

### 量价背离（警告信号，不是立刻行动信号）
- 价格创新高+成交量未创新高 = 量价顶背离 = 顶部警告
- 价格创新低+成交量未创新低 = 量价底背离 = 底部警告
- **背离只是警告**：顶背离后等价格跌破关键支撑才确认，底背离后等价格突破关键阻力才确认

## 六、失败条件（结构/事件无效判定）

### Spring失败条件
| 规则ID | 条件 | 含义 |
|--------|------|------|
| FAIL-SP-01 | 跌破时成交量放大（≥SC量） | 供应没有枯竭，不是有效Spring |
| FAIL-SP-02 | 跌破后没有迅速反弹 | 需求未进场，Spring失败 |
| FAIL-SP-03 | 反弹无力（缩量短阳） | 需求不足 |
| FAIL-SP-04 | 价格再次跌破Spring低点且放量 | Spring失败，底部不成立，必须止损 |
| FAIL-SP-05 | 1号Spring（跌破+高量） | 风险高，不理想的买入信号 |

### 底部/吸筹结构失败条件
| 规则ID | 条件 | 含义 |
|--------|------|------|
| FAIL-BT-01 | 连续ST中低点逐次降低 | 支撑在失效 |
| FAIL-BT-02 | 连续ST中成交量不减反增 | 供应增加，底部可能失败 |
| FAIL-BT-03 | ST后反弹无力 | 需求不足 |
| FAIL-BT-04 | SOS回调时成交量大或幅度大 | SOS可能是假信号 |

### UTAD失败条件（真突破而非假突破）
- 突破后能持续上涨，回测不破 → 是真突破不是UTAD
- 成交量支持突破方向 → 不是假突破
- **如果UTAD表现为真突破 → 派发结构可能不成立**

### 停止行为失败
- 停止行为后价格继续朝不利方向移动且成交量放大 → 停止行为无效

### 信号无效的通用规则
- 没有SC/AR背景的长下影线 → 不可信
- 长下影线+成交量不大 → 没有大规模需求进场
- 长下影线后第二天低开或下跌 → 长下影线无效
- 上影线后第二天继续上涨 → 上影线被消化（无效）
- 上影线后第二天下跌或低开 → 上影线有效
- **孤立看K线形态是散户被套的常见原因——必须看背景**

### 假突破vs真突破判定
| | 假突破（Spring/UTAD） | 真突破（SOS/SOW） |
|------|------|------|
| 持续性 | 无法持续，快速收回 | 能持续，回测不破 |
| 成交量 | 不支持突破方向 | 支持突破方向 |
| K线 | 反转形态（长影线） | 延续形态 |
| 状态机用途 | Spring/UTAD必须通过假突破验证 | SOS/SOW必须通过真突破验证 |

## 七、结构性规则（TR/Creek/Ice/死角/冰线）

### 震荡区间（TR）规则
- **区间形成**：SC/BC确立下/上边界，AR确立对侧边界
- **区间确认**：至少有两次高点和两次低点确认边界
- **区间需有足够幅度**，太窄不值得交易
- **区间内交易**：下边界做多，上边界做空/离场
- **出现SOS或SOW → 区间可能被打破**，停止区间交易

### Creek（小溪）与 Ice（冰线）
- **Creek**：吸筹区间内的阻力线，连接反弹高点。SOS突破Creek后，Creek变为支撑（LPS回测参考线）
- **Ice**：派发区间内的支撑线，连接回调低点。SOW跌破Ice后，Ice变为阻力（LPSY反弹压制线）
- **冰线**（持仓管理概念）：上涨趋势中最后一个明显回调低点。跌破后变阻力，上方有大量套牢盘形成抛压

### 支撑/阻力转换规则
- 支撑被有效跌破后 → 转变为阻力
- 阻力被有效突破后 → 转变为支撑
- **强支撑/阻力判定**：多次测试未破+测试时量能明显变化+持续时间长+形成区间
- **弱支撑/阻力判定**：仅测试一两次+量能无变化+时间短

### 死角（Hinge/Apex）规则
- **死角特征**：高点不断降低+低点不断升高+振幅缩小
- 死角意味着蓄力，即将出现大幅波动
- **死角突破必须放量确认**，止损在相反边界
- 完整序列：SC→AR→ST→死角→突破→SOS→JOC

### 50%回调规则
- 回调到前一波上涨的50%位置+缩量企稳 → 健康回调/加仓点
- 跌破50%位置继续下跌 → 回调可能变反转，应离场

### 高支撑规则
- 每次回调低点越来越高 → 需求越来越积极（高支撑信号）
- 高支撑是趋势健康的信号，也是趋势即将加速的信号

### 吸收行为
- **定义**：成交量放大（供应释放）+价格下跌有限或迅速反弹（需求承接）
- **三种表现**：长下影线 / 威胁性阴线后不跟随 / 回调到支撑位企稳
- 吸收行为持续出现 → 继续持有；吸收行为消失 → 趋势可能终止

## 八、多时间框架约束

### 多时间框架层级规则
- 周线/慢速图定方向（大趋势）— **主导趋势由慢速图定义，不可违反**
- 日线/中速图定策略（波段操作）
- 小时线/快速图定入场（精确时机）

### 冲突解决
- 更大时间框架的信号**优先级更高**
- 当不同时间框架信号冲突时，**服从更大框架**
- 例：H4显示吸筹完成但D1显示派发进行中 → 服从D1，降低H4信号置信度

### 因果传导约束
- 变化必须从快速图→中速图→慢速图依次传导，才能发展为长期趋势
- 整理震荡时间越长，后续趋势反转越显著且持久（因果定律应用）
- 顺大势交易→风险低，可长期持有；逆大势交易→风险高，只能短线

### 多维度信号交叉验证（九步检测法前4步）
1. 大盘地位判断（必须确定牛/熊/横盘）— 权重40%
2. 板块相对强度（必须>1做多）— 权重30%
3. 个股相对强度（必须属板块内最强）— 权重30%
4. 吸筹/派发阶段是否完成（区间足够长=充分的因）
- **逆大盘操作胜率极低，必须避免**
- **九步中任何一步未通过，都不执行交易**

## 九、测试门控规则（供给测试/需求测试）

### 供给测试（吸筹阶段后，验证卖压是否被消除）
- **目的**：确认原先有大量卖盘的价格区间是否还有卖压
- **方法**：利用利空消息使价格跌回原先有大量卖盘的区域
- **成功条件**：低成交量 → 卖压已被消除 → 可以推高市场 → 进入上涨趋势
- **失败条件**：高成交量 → 卖方卷土重来 → 需要继续震仓 → **回到吸筹**
- **K线特征**：低实体，伴随长下影线，可以是阳线或阴线

### 需求测试（派筹阶段后，验证买压是否被满足）
- **目的**：确认原先有大量买盘的价格区间是否还有买压
- **方法**：利用利好消息将价格推高回原先有大量买盘的区域
- **成功条件**：低成交量 → 买压已被满足 → 可以推动下跌 → 进入下跌趋势
- **失败条件**：高成交量 → 买方涌入 → 需要继续派筹 → **回到派筹**

### 测试通用约束
- 测试是最强烈的信号之一——局内人发出的清晰信号，表明市场即将突破
- **测试是门控条件**：测试通过才能进入下一阶段，测试失败则循环回前一阶段
- 成功测试 = 低成交量 = 反向力量已枯竭
- 失败测试 = 高成交量 = 反向力量仍存在

### 连续测试的健康/不健康判定
| | 健康（结构巩固） | 不健康（结构失败） |
|------|------|------|
| 低点 | 逐渐升高或持平 | 逐次降低 |
| 成交量 | 逐次递减 | 不减反增 |
| 反弹 | 每次都有反弹 | 反弹无力 |
| 结论 | 供应逐步枯竭 | 底部可能失败 |

## 十、待定决策的理论回答

基于理论文档，回答 `cognitive-architecture-decisions.md` 中的4个待定决策：

### 待定1回答：每个阶段的必要/充分条件

**A阶段（PS/SC/AR/ST）— 停止行为阶段**：
- PS可选（有些底部没有明显PS）
- **SC必须出现**（没有SC，底部缺乏供应消耗）
- **AR必须出现**（没有AR，说明需求不足，无法形成区间）
- **ST必须出现且成交量<SC**（没有ST，底部不稳固）
- 四个事件缺一不可（PS/SC/AR/ST），没有PS只是第一个信号弱一些

**B阶段 — 横向吸筹/派发主体**：
- 理论没有给出具体的K线数下限
- 但因果定律要求：**横向整理时间越长，后续趋势越强**
- 区间不够长 → 后续SOS/SOW可信度降低
- 吸筹阶段特征：下跌时缩量（抛压枯竭）+反弹时温和放量（资金承接）+不创新低
- 派发阶段特征：上涨时缩量（买盘不足）+下跌时放量（资金出逃）+不创新高
- **建议约束**：B阶段至少应包含2-3个完整的震荡周期（高-低-高或低-高-低）

**C阶段（Spring/UTAD）— 最终测试**：
- Spring/UTAD**不是必须出现**的，但如果出现是最强信号之一
- 有些吸筹没有Spring（直接从B阶段过渡到D阶段），此时用LPS替代
- 有些派发没有UTAD（直接SOW），说明主力出货更急迫
- **3号Spring**（仅测试支撑但未跌破）也算有效但较弱

**D阶段（SOS/SOW）— 确认阶段**：
- SOS/SOW与B阶段的区分标准：**放量突破Creek/Ice**
- B阶段的反弹/回调不突破Creek/Ice，D阶段的SOS/SOW必须突破
- SOS后回调缩量=真SOS；SOS后回调放量=假SOS（仍在B阶段）

### 待定2回答：结构失败的完整判定标准

根据理论文档，结构失败的判定：
1. **跌破SC_LOW且放量** → Spring失败 → 底部不成立
2. **连续ST低点逐次降低** → 支撑在失效
3. **连续ST成交量不减反增** → 供应增加
4. **SOS/SOW后回调放量** → 假信号
5. **供给测试失败（高成交量）** → 回到吸筹/派发重新开始
- 失败后应该：**回到TRENDING模式**，等待新的停止行为
- 不是"开始新结构"——必须先回到趋势模式再重新观察

### 待定3回答：TRANSITIONING的完整路径

根据理论文档的事件序列和测试门控：
1. 停止行为 → AR → ST不破 → **区间假设成立** → 进入RANGING
2. 停止行为 → AR → ST破了（成交量大） → **供给测试失败** → 回到TRENDING（等新的停止行为）
3. 停止行为 → 没有明显AR → **需求不足，不构成区间** → 回到TRENDING
4. 停止行为 → AR → ST不破 → 但后续连续测试低点降低+量增 → **结构逐步恶化** → 回到TRENDING
- **关键原则**：测试是门控条件，测试失败=循环回前一阶段

### 待定4回答：区间斜率的处理

根据理论文档：
- SC→ST连线可能有斜率（底部抬升=吸筹信号更强）
- 区间可以上斜（higher lows=需求积极）、水平、下斜
- **高支撑规则**：每次回调低点越来越高 → 需求越来越积极 → 趋势即将加速
- 斜率影响后续判断：上斜区间 → 吸筹信号更强 → 置信度上升
- 理论没有给出精确的斜率计算公式，但提供了定性判断：
  - 连接SC_LOW和后续ST的低点，观察趋势
  - 平行线经过AR附近（可以不精确穿过AR_HIGH，大致平行即可）
