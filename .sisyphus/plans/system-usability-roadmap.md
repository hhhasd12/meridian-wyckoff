# 威科夫量化系统 — 从"能跑"到"可用"的待解决问题清单

> 背景：v3.0 引擎+进化系统代码层面已完成（1271 tests passing），但作为一个**可使用的量化系统**，存在大量产品层面的缺失。本文档记录所有已知问题和待探讨方向。

## 一、可观测性（看不见系统在做什么）

### 1.1 进化过程不可视
- **问题**：进化在后台跑，前端只有一个 fitness 曲线和基础状态面板。完全看不到：
  - 每个个体的具体参数是什么
  - 回测过程中在哪里开仓/平仓了
  - 威科夫状态机在历史数据上标注了什么阶段（吸筹/上涨/派发/下跌）
  - 标注是否正确、区间画得准不准
  - 哪些交易赚了、哪些亏了、为什么
- **期望**：能在前端看到进化最优个体在历史K线上的完整回测轨迹——开平仓标记、威科夫阶段标注、TR区间、FVG标记等

### 1.2 实盘信号不可追溯
- **问题**：实盘运行时，只能看到"买/卖/等待"信号，看不到信号产生的推理过程
- **期望**：每个信号应该展示——哪些因子支持、哪些反对、置信度分解、多周期各自的判断

### 1.3 状态机可视化缺失
- **问题**：威科夫状态机是系统核心，但前端没有任何状态机的可视化
- **期望**：能看到当前处于哪个阶段、转换条件是什么、距离下个阶段还差什么

## 二、系统可控性（控不住系统行为）

### 2.1 热更新机制缺失
- **致命问题**：进化跑了几个小时，中途改了后端代码，要么：
  - 放弃进化进度重启 → 浪费几小时
  - 不重启但新代码不生效 → 改了等于没改
  - 前端断开连接 → 不知道进化还在不在跑
- **期望**：
  - 进化进度可持久化（中断恢复）
  - 后端代码热重载不影响正在运行的进化
  - 前端断开重连后能恢复状态

### 2.2 进化结果→实盘的闭环缺失
- **现状**：进化完成后，best_config 存在 JSON 文件里，需要手动复制到 config.yaml 才能应用到实盘
- **期望**：前端一键"应用此配置到实盘"（带确认），或者至少有个对比视图（当前配置 vs 进化推荐配置）

### 2.3 参数不可调
- **问题**：进化的超参数（种群大小、变异率、数据范围等）只能改代码或 config.yaml，前端无法调整
- **期望**：前端可配置进化参数，实时生效

## 三、数据完整性

### 3.1 数据自动更新
- **问题**：8年历史数据是用 `fetch_data.py` 手动拉取的，没有自动增量更新
- **期望**：系统启动时或定时自动拉取最新数据

### 3.2 多币对支持
- **现状**：只有 BTC/USDT，代码里硬编码了很多 BTC 相关逻辑
- **期望**：至少支持 ETH/USDT 等主流币对

## 四、交易执行

### 4.1 实盘执行链路未验证
- **现状**：exchange_connector 有代码但从未在真实交易所上执行过
- **关键问题**：
  - 订单类型（限价/市价）选择
  - 滑点处理
  - 部分成交处理
  - 网络异常重试
  - 资金管理（每次投入多少比例）

### 4.2 风控缺失
- **问题**：有 risk_management 和 circuit_breaker 插件，但缺乏：
  - 日亏损上限
  - 单笔最大仓位限制
  - 连续亏损后自动暂停
  - 异常行情保护（闪崩/插针）

## 五、前端功能缺口

### 5.1 已有但不够
- K线图：有，但没有威科夫标注叠加
- 进化面板：有，但只有 fitness 曲线，没有回测详情
- 状态栏：有，但信息太少

### 5.2 完全缺失
- 回测可视化（历史K线 + 开平仓标记 + 威科夫阶段标注）
- 策略对比（不同进化代的参数对比）
- 交易日志（每笔交易的详细记录）
- 账户概览（余额、持仓、盈亏曲线）
- 通知配置（Telegram 通知的开关和频率）

## 六、部署和运维

### 6.1 Docker 部署
- 有 Dockerfile 但未充分测试
- 需要确认：数据卷挂载、日志持久化、配置注入

### 6.2 监控告警
- 系统健康检查有 health_check.py 但没有持续监控
- 进化异常终止没有通知机制

---

## 优先级建议

| 优先级 | 问题 | 理由 |
|--------|------|------|
| P0 | 热更新/进化中断恢复 | 每次代码修改都浪费几小时进化时间 |
| P0 | 回测可视化（前端） | 不能看就不能判断进化结果是否可信 |
| P1 | 进化结果→实盘闭环 | 进化完了不能用等于白跑 |
| P1 | 威科夫状态机可视化 | 核心逻辑的可观测性 |
| P2 | 实盘执行链路验证 | 真正要交易时必须的 |
| P2 | 风控完善 | 保护资金安全 |
| P3 | 数据自动更新 | 运维效率 |
| P3 | 多币对 | 功能扩展 |
